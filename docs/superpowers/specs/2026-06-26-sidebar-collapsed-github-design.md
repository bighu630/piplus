# Sidebar 折叠态与 GitHub 入口设计

**日期：** 2026-06-26
**范围：** `apps/web/src/components/Sidebar.tsx`

## 背景
当前左侧边栏在折叠态仍保留较多元素语义（项目文件夹图标、底部功能按钮区域），不符合“折叠后仅作为展开入口”的使用目标。同时，展开态底部缺少跳转到项目 GitHub 仓库的快捷入口。

## 目标
- 在展开态底部设置按钮左侧新增 GitHub 图标按钮，点击跳转到本项目 GitHub 仓库。
- 将折叠态简化为仅承担“展开侧边栏”的入口，不在折叠态中承载项目选择或底部功能操作。

## 交互设计
### 展开态
- 保持现有信息架构：标题、新建项目、搜索、过滤器、项目树、会话树、底部操作区。
- 在底部操作区中，将 GitHub 图标按钮放置在设置按钮左侧。
- GitHub 按钮点击后在新标签页打开 `https://github.com/bighu630/piplus`，使用 `noopener,noreferrer`。

### 折叠态
- 仅显示一个顶部展开按钮。
- 主体区域仅显示各个项目的首字母徽标。
- 不显示：新建项目按钮、搜索框、过滤器、会话树、GitHub、设置、退出登录。
- 点击顶部展开按钮或任一项目首字母，行为一致：仅展开侧边栏，不触发项目选择、不触发会话切换。

## 视觉设计
- 项目首字母取 `project.name.trim()[0]`，并对英文字符执行大写转换；中文等非英文字符保留原字符。
- 首字母采用紧凑的圆角徽标样式，与现有蓝灰色界面保持一致，不引入新的主题色系统。
- 折叠态维持当前窄宽度，不调整全局布局尺寸。

## 实现策略
- 仅修改 `apps/web/src/components/Sidebar.tsx`。
- 在组件内部增加一个轻量辅助函数，生成项目展示字母。
- 通过 `isSidebarCollapsed` 分支，分别渲染展开态和折叠态的主体区与底部区。
- 保持现有 props 接口不变，避免影响 `apps/web/src/App.tsx` 或其他父组件。

## 错误处理与兼容性
- 若项目名为空字符串或仅包含空白字符，回退显示 `?`。
- GitHub 跳转使用 `window.open`；若浏览器阻止弹窗，则不额外增加错误提示，遵循当前简洁交互风格。

## 测试要点
- 展开态下 GitHub 按钮存在且位置位于设置按钮左侧。
- 点击 GitHub 按钮触发正确仓库地址跳转。
- 折叠态下仅可见展开按钮和项目首字母，不显示其他操作元素。
- 点击折叠态项目首字母仅展开侧边栏，不触发项目选择。
